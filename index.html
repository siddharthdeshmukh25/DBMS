P2
ER Diagram – Roadway Travel Management System
▶
🎯 Purpose

Design an ER diagram to model real-world entities and relationships, then implement as SQL tables.

Entities

  • Customer: CustomerID (PK), Name, Phone, Email
  • TravelRoute: RouteID (PK), Source, Destination, Distance
  • Booking: BookingID (PK), CustomerID (FK), RouteID (FK), Date, Seats

SQL Code

CREATE TABLE Customer (
  CustomerID INT PRIMARY KEY,
  Name VARCHAR(60), Phone VARCHAR(15), Email VARCHAR(80)
);
CREATE TABLE TravelRoute (
  RouteID INT PRIMARY KEY,
  Source VARCHAR(50), Destination VARCHAR(50), Distance FLOAT
);
CREATE TABLE Booking (
  BookingID INT PRIMARY KEY,
  CustomerID INT, RouteID INT, BookDate DATE, Seats INT,
  FOREIGN KEY(CustomerID) REFERENCES Customer(CustomerID),
  FOREIGN KEY(RouteID) REFERENCES TravelRoute(RouteID)
);
INSERT INTO Customer VALUES(1,'Sneha','9876543210','sneha@mail.com');
INSERT INTO TravelRoute VALUES(1,'Pune','Mumbai',148.5);
INSERT INTO Booking VALUES(1,1,1,'2026-04-06',2);

▶
🎯 Purpose

SQL Code

CREATE TABLE Department (
  DeptID   INT PRIMARY KEY,

);
CREATE TABLE EmpInfo (
  EmpID  INT PRIMARY KEY,

P5
Integrity Constraints – NOT NULL, CHECK, DEFAULT
▶
🎯 Purpose

NOT NULL prevents empty values, CHECK validates data range, DEFAULT fills missing values automatically.

SQL Code

CREATE TABLE CustomerDetails (
  CustID INT         PRIMARY KEY,
  Name   VARCHAR(80) NOT NULL,
  Age    INT         CHECK(Age >= 18 AND Age <= 100),
  City   VARCHAR(50) DEFAULT 'Pune'
);
-- Valid: City defaults to Pune
INSERT INTO CustomerDetails(CustID,Name,Age) VALUES(1,'Anjali',25);
-- NOT NULL violation (ERROR)
INSERT INTO CustomerDetails VALUES(2,NULL,30,'Mumbai');
-- CHECK violation: 15 < 18 (ERROR)
INSERT INTO CustomerDetails VALUES(3,'Raj',15,'Nashik');
SELECT * FROM CustomerDetails;

P7
Normalization – 1NF, 2NF, 3NF, BCNF
▶
🎯 Purpose

Eliminate data redundancy and anomalies by decomposing an unnormalized table step-by-step.

Quick Reference

  • 1NF: Atomic values, no repeating groups
  • 2NF: 1NF + No partial dependencies
  • 3NF: 2NF + No transitive dependencies
  • BCNF: Every determinant is a candidate key

SQL Code

-- 1NF: Atomic values per cell
CREATE TABLE Orders_1NF (
  OrderID INT, CustName VARCHAR(60), Item VARCHAR(50),
  PRIMARY KEY(OrderID, Item)
);
-- 2NF: Remove partial dependency
CREATE TABLE Customers_2NF (CustID INT PRIMARY KEY, CustName VARCHAR(60));
CREATE TABLE OrderItems_2NF (
  OrderID INT, CustID INT, Item VARCHAR(50),
  PRIMARY KEY(OrderID,Item),
  FOREIGN KEY(CustID) REFERENCES Customers_2NF(CustID)
);
-- 3NF: Remove transitive dependency
CREATE TABLE ZipCity (ZipCode VARCHAR(10) PRIMARY KEY, City VARCHAR(50));
P8
Aggregate Functions – COUNT, SUM, AVG, MIN, MAX
▶
🎯 Purpose

Use aggregate functions with GROUP BY and HAVING to summarize and analyze sales data.

SQL Code

CREATE TABLE Sales (
  SaleID INT PRIMARY KEY, Product VARCHAR(50),
  Region VARCHAR(30), Amount DECIMAL(10,2), SaleDate DATE
);
INSERT INTO Sales VALUES
(1,'Laptop','West',55000,'2026-01-10'),
(2,'Phone','North',22000,'2026-01-15'),
(3,'Tablet','West',18000,'2026-02-05'),
(4,'Laptop','South',60000,'2026-02-20');
SELECT COUNT(*) AS Total FROM Sales;
SELECT SUM(Amount) AS Revenue FROM Sales;
SELECT AVG(Amount) AS AvgSale FROM Sales;
SELECT MIN(Amount), MAX(Amount) FROM Sales;
SELECT Region, SUM(Amount) FROM Sales GROUP BY Region HAVING SUM(Amount) > 50000;
P9
PL/SQL – Calculate Grade of Minimum 10 Students
▶
🎯 Purpose

Write a PL/SQL block using cursor and loop to assign grades based on marks for 10+ students.

SQL Code

CREATE TABLE Students (
  RollNo INT PRIMARY KEY, Name VARCHAR(60), Marks INT, Grade CHAR(2)
);
INSERT INTO Students(RollNo,Name,Marks) VALUES
(1,'Aarav',92),(2,'Bela',78),(3,'Chetan',85),
(4,'Divya',61),(5,'Eshan',45),(6,'Farida',73),
(7,'Gaurav',88),(8,'Hina',55),(9,'Isha',96),(10,'Jay',67);

DECLARE
  CURSOR c1 IS SELECT RollNo, Marks FROM Students;
  v_roll  Students.RollNo%TYPE;
  v_marks Students.Marks%TYPE;
  v_grade CHAR(2);
BEGIN
  OPEN c1;
  LOOP
    FETCH c1 INTO v_roll, v_marks;
    EXIT WHEN c1%NOTFOUND;
    IF    v_marks>=90 THEN v_grade:='A+';
    ELSIF v_marks>=80 THEN v_grade:='A';
    ELSIF v_marks>=70 THEN v_grade:='B';
    ELSIF v_marks>=60 THEN v_grade:='C';
    ELSE                   v_grade:='F';
    END IF;
    UPDATE Students SET Grade=v_grade WHERE RollNo=v_roll;
  END LOOP;
  CLOSE c1;
END;
/
SELECT * FROM Students;
P10
MongoDB – Design & Implement 5 Queries
▶
🎯 Purpose

Understand NoSQL concepts with MongoDB — insert documents and run 5 different query types.

MongoDB Commands

use schoolDB

db.students.insertMany([
  { name:"Aisha",  age:20, marks:88, city:"Pune"   },
  { name:"Rohan",  age:21, marks:74, city:"Mumbai" },
  { name:"Meera",  age:19, marks:92, city:"Nagpur" },
  { name:"Vikram", age:22, marks:61, city:"Pune"   },
  { name:"Pooja",  age:20, marks:79, city:"Nashik" }
])
// Q1: Find all
db.students.find()
// Q2: From Pune only
db.students.find({ city:"Pune" })
// Q3: Marks greater than 80
db.students.find({ marks:{ $gt:80 } })
// Q4: Update Rohan's marks
db.students.updateOne({ name:"Rohan" },{ $set:{ marks:80 } })
// Q5: Delete students with marks < 65
db.students.deleteMany({ marks:{ $lt:65 } })

SQL vs MongoDB

  • Tables/Rows → Collections/Documents
  • Fixed schema → Flexible JSON-like (BSON)
  • SELECT → find()
  • INSERT → insertOne/insertMany
